What is Ansible ?
Ansible is an open-source IT engine which automates the IT tools such as intra service orchestration, application deployment, cloud provisioning, etc.

Ansible is easy to deploy because it does not use any agents or custom security infrastructure on the client-side, and by pushing modules to the clients. These modules are executed locally on the client-side, and the output is pushed back to the Ansible server.

It can easily connect to clients using SSH-Keys, simplifying though the whole process. Client details, such as hostnames or IP addresses and SSH ports, are stored in the files, which are called inventory files. If you created an inventory file and populated it, then Ansible can use it.

Why Use Ansible ?

  • Ansible is free to use by everyone.
  • Ansible is very consistent and lightweight, and no constraints regarding the OS are present.
  • It is very secure due to its agentless capabilities and open SSH security features.
  • Ansible does not need any special system administrator skills to install and use it.
  • Ansible has a comprehensive documentation and easy to learn structure and configuration.

    • Ansible Workflow
    Ansible works by connecting to your nodes and pushing out a small program called Ansible modules to them. Then Ansible executed these modules and removed them after finished. The library of modules can reside on any machine, and there are no daemons, servers, or databases required.



    Terminologies

    Control Node
    It can be called as main machine/server through which we run ansible commands on he machone we need to manage.

    Managed Node
    These are machines we manage through Ansible on control node. These are by default managed using the SSH network protocol to ensure that machines are in required state.

    ansible.cfg
    Manages the behaviour of an ansible installation.

    Hosts
    In the Ansible architecture, hosts are the node systems, which are automated by Ansible, and any machine such as RedHat, Linux, Windows, etc.

    Inventory
    Collection of IP/hostnames of all the managed nodes. These hosts can be combined as a group or individually depending on our requirements.

    Module
    The module is a command or set of similar commands which is executed on the client-side.

    Task
    A task is a section which consists of a single procedure to be completed.

    Role
    It is a way of organizing tasks and related files to be later called in a playbook.

    Fact
    The information fetched from the client system from the global variables with the gather facts operation.

    Play
    It is the execution of the playbook.

    Tag
    It is a name set to a task that can be used later on to issue just that specific task or group of jobs.

    Ad-hoc commands
    Ad-hoc commands are one of the simplest ways of using Ansible. These are used when you want to issue some commands on a server or bunch of servers. The ad-hoc commands are not stored for future use, but it represents a fast way to interact with the desired servers.

    The Ansible ad-hoc command uses the /usr/bin/ansible command-line tool to automate a single task on one or more managed nodes. The Ad-hoc commands are quick and easy, but they are not re-usable. The Ad-hoc commands demonstrate the simplicity and power of Ansible.
